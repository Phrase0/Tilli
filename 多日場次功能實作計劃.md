# PocketPOS 多日場次功能實作計劃

## 🎯 總體目標
將現有的單日場次系統升級為支援多日/跨日場次的完整解決方案，包含資料模型調整、UI/UX 改進和報表功能增強。

## 🔧 技術細節補充

### 建議的 CDSessionEntity 新欄位
```swift
@NSManaged public var startDate: Date      // 場次開始日期
@NSManaged public var endDate: Date        // 場次結束日期  
@NSManaged public var isMultiDay: Bool     // 是否為多日場次
@NSManaged public var isAllDay: Bool       // 是否為全天場次
@NSManaged public var timezone: String?    // 時區資訊（可選）

// 保留向後相容性
@NSManaged public var date: Date           // 作為 displayDate 使用
```

### 場次驗證邏輯
```swift
enum SessionValidationResult {
    case valid
    case warning(message: String, suggestedSessions: [Session])
    case error(message: String)
}

func validateTransaction(_ transaction: Transaction, for session: Session) -> SessionValidationResult
```

### 日曆顯示樣式
- **單日場次**: 藍色圓點
- **多日場次**: 連續藍色條帶
- **虛擬場次**: 灰色淡化樣式
- **混合日期**: 圓點 + 條帶組合

## ⚠️ 注意事項

1. **資料一致性**: 確保所有新舊場次資料格式一致
2. **效能考量**: 多日場次查詢可能影響效能，需要適當的索引和快取
3. **使用者體驗**: 每個階段都要確保現有功能正常運作
4. **測試覆蓋**: 每個 Phase 完成後都要進行完整測試

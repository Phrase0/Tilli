# PocketPOS 多日場次功能實作計劃

## 🎯 總體目標
將現有的單日場次系統升級為支援多日/無限期場次的完整解決方案，包含資料模型調整、UI/UX 改進和報表功能增強。

---

## 📋 核心設計決策

### 場次與交易的關係定義

**核心原則：場次 = 標籤/分類，不是嚴格的時間範圍限制**

* **場次設定日期**：主要用於日曆顯示和識別市集活動
* **交易歸屬**：交易掛上場次標籤即屬於該場次，不受場次設定日期限制
* **報表統計**：基於實際交易日期，不是場次設定日期

**範例說明：**
```
場次設定：春節市集 2025/01/15（單日）
實際交易：1/14, 1/15, 1/16 都有交易

結果：
• 這些交易都屬於「春節市集」
• 報表統計包含 1/14-1/16 的所有交易
• 不顯示警告、不需要使用者處理
• 場次設定日期僅用於日曆識別
```

## 📊 場次類型定義

### 三種場次類型

#### 1. 單日場次
* **設定**：2025/01/15
* **用途**：單日市集、快閃活動
* **日曆顯示**：單個藍色圓點
* **報表時間**：固定為該日

#### 2. 多日場次
* **設定**：2025/01/15 - 2025/01/20
* **用途**：連續多天的市集活動
* **日曆顯示**：每天藍色圓點（連續顯示）
* **報表時間**：可選「全部」或「自訂範圍」

#### 3. 無限期場次
* **設定**：2025/01/15 起
* **用途**：常態營運攤位、長期場次
* **日曆顯示**：
  - 在 startDate 當天顯示小圓點
  - 從 startDate 開始（含當天），場次固定顯示在日曆下方欄位 + ∞ 符號
  - startDate 之前不顯示
* **報表時間**：必須選擇範圍（最近 30 天/自訂，**最多 90 天**）

```swift
enum SessionDateType: String, Codable {
    case single      // 單日
    case multi       // 多日
    case permanent   // 無限期
}

struct SessionModel {
    var id: UUID
    var title: String
    var startDate: Date              // 必填
    var endDate: Date?               // nil = 無限期
    var dateType: SessionDateType
    var categories: [CategoryModel]
    var createdAt: Date
    var currency: String = "TWD"

    // 計算屬性
    var displayDate: Date { startDate }

    var displayDateRange: String {
        switch dateType {
        case .single:
            return formatDate(startDate)
        case .multi:
            return "\(formatDate(startDate)) - \(formatDate(endDate!))"
        case .permanent:
            return "\(formatDate(startDate)) 起"
        }
    }

    var dayCount: Int? {
        guard let end = endDate else { return nil }
        return Calendar.current.dateComponents([.day], from: startDate, to: end).day! + 1
    }

    var isActive: Bool {
        let today = Date()
        if let end = endDate {
            return today >= startDate && today <= end
        } else {
            return today >= startDate  // 無限期永遠 active
        }
    }

    var status: SessionStatus {
        let today = Calendar.current.startOfDay(for: Date())
        let start = Calendar.current.startOfDay(for: startDate)

        switch dateType {
        case .single, .multi:
            if let end = endDate {
                let endDay = Calendar.current.startOfDay(for: end)
                if today < start { return .upcoming }
                else if today > endDay { return .completed }
                else { return .ongoing }
            }
        case .permanent:
            if today < start { return .upcoming }
            else { return .ongoing }  // 永遠不會變成 completed
        }
        return .upcoming
    }
}
```

func fetchTransactions(forSessionId id: UUID, dateRange: DateInterval?) -> [TransactionModel] {
    let fetchRequest: NSFetchRequest<CDTransactionEntity> = CDTransactionEntity.fetchRequest()

    var predicates: [NSPredicate] = [
        NSPredicate(format: "sessionId == %@", id as CVarArg)
    ]

    if let range = dateRange {
        predicates.append(
            NSPredicate(format: "timestamp >= %@ AND timestamp <= %@",
                       range.start as CVarArg,
                       range.end as CVarArg)
        )
    }

    fetchRequest.predicate = NSCompoundPredicate(andPredicateWithSubpredicates: predicates)
    fetchRequest.sortDescriptors = [NSSortDescriptor(keyPath: \CDTransactionEntity.timestamp, ascending: false)]

    do {
        let results = try context.fetch(fetchRequest)
        return results.map { $0.toModel() }
    } catch {
        print("Failed to fetch transactions: \(error)")
        return []
    }
}

```swift
// 判斷某日期是否有場次
func sessionsForDate(_ date: Date, from sessions: [SessionModel]) -> [SessionModel] {
    let day = Calendar.current.startOfDay(for: date)

    return sessions.filter { session in
        let start = Calendar.current.startOfDay(for: session.startDate)

        switch session.dateType {
        case .single:
            return day == start

        case .multi:
            guard let endDate = session.endDate else { return false }
            let end = Calendar.current.startOfDay(for: endDate)
            return day >= start && day <= end

        case .permanent:
            return day == start  // 只在開始日期顯示圓點
        }
    }
}

// 獲取所有永久場次（固定顯示在日曆下方）
// 只返回 startDate <= 當前選中日期的永久場次
func getPermanentSessions(from sessions: [SessionModel], selectedDate: Date) -> [SessionModel] {
    let selectedDay = Calendar.current.startOfDay(for: selectedDate)
    return sessions.filter { session in
        guard session.dateType == .permanent else { return false }
        let startDay = Calendar.current.startOfDay(for: session.startDate)
        return startDay <= selectedDay  // 只顯示已經開始的永久場次
    }
}

// 場次資訊顯示
func sessionInfo(for session: SessionModel, on date: Date) -> String {
    switch session.dateType {
    case .single:
        return session.title

    case .multi:
        guard let endDate = session.endDate else { return session.title }
        let start = Calendar.current.startOfDay(for: session.startDate)
        let current = Calendar.current.startOfDay(for: date)
        let dayNumber = Calendar.current.dateComponents([.day], from: start, to: current).day! + 1
        let totalDays = Calendar.current.dateComponents([.day], from: start, to: endDate).day! + 1
        return "\(session.title) - 第 \(dayNumber) 天/共 \(totalDays) 天"

    case .permanent:
        let start = Calendar.current.startOfDay(for: session.startDate)
        let current = Calendar.current.startOfDay(for: date)
        let daysSinceStart = Calendar.current.dateComponents([.day], from: start, to: current).day! + 1
        return "\(session.title) - 開始至今第 \(daysSinceStart) 天"
    }
}
```

struct ReportTimeRange {
    var type: RangeType
    var customStart: Date
    var customEnd: Date
    let session: SessionModel

    enum RangeType {
        case all            // 全部
        case recent30       // 最近30天
        case custom         // 自訂
    }

    var actualStart: Date {
        switch type {
        case .all:
            return session.startDate
        case .recent30:
            return Calendar.current.date(byAdding: .day, value: -29, to: Date())!
        case .custom:
            return customStart
        }
    }

    var actualEnd: Date {
        switch type {
        case .all:
            return session.endDate ?? Date()
        case .recent30:
            return Date()
        case .custom:
            // 無限期場次限制最多90天
            if session.dateType == .permanent {
                let maxEnd = Calendar.current.date(byAdding: .day, value: 30, to: customStart)!
                return min(customEnd, maxEnd)
            }
            return customEnd
        }
    }

    var dayCount: Int {
        Calendar.current.dateComponents([.day], from: actualStart, to: actualEnd).day! + 1
    }

    var dateInterval: DateInterval {
        DateInterval(start: actualStart, end: actualEnd)
    }
}

## 🔧 技術實作細節

### 場次類型定義

```swift
enum SessionDateType: String, Codable {
    case single      // 單日
    case multi       // 多日
    case permanent   // 無限期
}

struct SessionModel {
    var id: UUID
    var title: String
    var startDate: Date              // 必填
    var endDate: Date?               // nil = 無限期
    var dateType: SessionDateType

    // 計算屬性
    var displayDate: Date { startDate }

    var displayDateRange: String {
        switch dateType {
        case .single:
            return formatDate(startDate)
        case .multi:
            return "\(formatDate(startDate)) - \(formatDate(endDate!))"
        case .permanent:
            return "\(formatDate(startDate)) 起"
        }
    }

    var isActive: Bool {
        let today = Date()
        if let end = endDate {
            return today >= startDate && today <= end
        } else {
            return today >= startDate  // 無限期永遠 active
        }
    }
}
```

### 報表時間範圍

```swift
struct ReportTimeRange {
    var type: RangeType
    var customStart: Date
    var customEnd: Date
    let session: SessionModel

    enum RangeType {
        case all            // 全部
        case recent30       // 最近30天
        case custom         // 自訂
    }

    var actualStart: Date {
        switch type {
        case .all:
            return session.startDate
        case .recent30:
            return Calendar.current.date(byAdding: .day, value: -29, to: Date())!
        case .custom:
            return customStart
        }
    }

    var actualEnd: Date {
        switch type {
        case .all:
            return session.endDate ?? Date()
        case .recent30:
            return Date()
        case .custom:
            // 無限期場次限制最多90天
            if session.dateType == .permanent {
                let maxEnd = Calendar.current.date(byAdding: .day, value: 30, to: customStart)!
                return min(customEnd, maxEnd)
            }
            return customEnd
        }
    }

    var dayCount: Int {
        Calendar.current.dateComponents([.day], from: actualStart, to: actualEnd).day! + 1
    }
}
```

### 報表資料查詢

```swift
class ReportDataManager {
    // 根據場次和時間範圍查詢交易
    func getTransactions(session: SessionModel, timeRange: ReportTimeRange) -> [TransactionModel] {
        return transactionDataManager.fetchTransactions(
            forSessionId: session.id,
            dateRange: timeRange.dateInterval
        )
    }

    // 每日營收統計
    func getDailyRevenue(session: SessionModel, timeRange: ReportTimeRange) -> [DailyRevenueData] {
        let transactions = getTransactions(session: session, timeRange: timeRange)

        let grouped = Dictionary(grouping: transactions) { transaction in
            Calendar.current.startOfDay(for: transaction.timestamp)
        }

        return grouped.map { date, txs in
            DailyRevenueData(
                date: date,
                amount: txs.reduce(0) { $0 + $1.totalAmount },
                count: txs.count
            )
        }.sorted { $0.date < $1.date }
    }
}
```

## ⚠️ 重要設計原則

### 1. 場次 = 標籤，不是時間限制
* 交易屬於哪個場次，由建立時選擇的場次決定
* 場次設定日期不限制交易日期
* 報表統計基於實際交易日期

### 2. 報表以實際資料為準
* 不顯示「超出範圍」警告
* 不強制修正場次日期
* 時段分佈、熱門商品等統計都基於實際交易時間

### 3. 使用者體驗優先
* 避免複雜的警告和修正機制
* 報表呈現清晰直觀
* 時間範圍選擇符合直覺

### 4. 效能考量
* 無限期場次強制選擇時間範圍且最多 90 天（避免查詢所有歷史資料）
* 長時間範圍的報表使用月度聚合
* 適當的資料快取機制
* Core Data 索引優化

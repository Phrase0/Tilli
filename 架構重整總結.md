# Core Data 架構重整總結

## 📌 新架構設計

### 1. 整體責任分層

#### SessionDataManager（主要入口）
- **職責**：Session 的 CRUD 操作 + Transaction 管理
- **特點**：
  - 作為入口，負責 fetch 整包 Session 與整體一致性檢查
  - CRUD 僅限於 Session 自身（新增、更新、刪除）
  - 管理 Transaction 的新增和更新（Transaction 永久保留）
  - 使用預先載入（prefetching）提高效能

#### CategoryRepository（分類專責）
- **職責**：Category 的專責 CRUD 操作
- **功能**：
  - 新增、更新、停用、啟用 Category
  - 智能刪除：有 Transaction 則停用，無則硬刪除
  - 自動檢查關聯的 Transaction 記錄

#### ProductRepository（產品專責）
- **職責**：Product 的專責 CRUD 操作
- **功能**：
  - 新增、更新、停用、啟用 Product
  - 智能刪除：有 Transaction 則停用，無則硬刪除
  - 保留業務邏輯的庫存更新規則

#### TransactionDataManager（查詢專用）
- **職責**：Transaction 的查詢和統計
- **特點**：
  - 僅提供查詢功能，不提供 CRUD
  - 添加統計方法（營收計算、熱銷商品等）
  - Transaction 的 CRUD 已移至 SessionDataManager

### 2. 軟刪除規則實現

#### Session
- ✅ **新增**：建立 Session 與其 categories、products
- ✅ **更新**：僅更新 Session 層級資訊（title, date…）
- ✅ **刪除**：硬刪除 Session，連帶刪除 Category/Product，但保留 Transaction

#### Category
- ✅ **新增**：建立 Category，掛在 Session 下
- ✅ **更新**：允許修改名稱等基本屬性
- ✅ **停用**：`isDisabled = true`，不影響 Product
- ✅ **啟用**：`isDisabled = false`
- ✅ **智能刪除**：
  - 有 Transaction → 禁止硬刪除，只能停用
  - 無 Transaction → 可刪除，同時刪除底下所有 Products

#### Product
- ✅ **新增**：建立 Product，掛在 Category 下
- ✅ **更新**：允許修改名稱、價格、庫存等屬性
- ✅ **停用**：`isDisabled = true`
- ✅ **啟用**：`isDisabled = false`
- ✅ **智能刪除**：
  - 有 Transaction → 禁止硬刪除，只能停用
  - 無 Transaction → 可刪除

#### Transaction
- ✅ **新增**：允許新增交易紀錄，永遠保留
- ✅ **更新**：僅允許修改業務欄位
- ❌ **刪除**：禁止刪除

### 3. 關鍵改進

#### 效能優化
- 使用 `relationshipKeyPathsForPrefetching` 避免多層查詢
- Repository 針對 CRUD 時，用最小範圍 fetch

#### 一致性檢查
- Category/Product 刪除前，必須查 Transaction
- 使用 JSON 解析檢查 Transaction items 中的 productId

#### 錯誤處理
- 提供明確的回傳結果類型（`CategoryDeletionResult`, `ProductDeletionResult`）
- UI 層可根據結果類型提供適當的使用者回饋
